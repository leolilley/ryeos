# Lift Terminators to Data (Deferred)

See `.tmp/STRATEGY-AND-GAPS/implementation-2-protocols-as-data/07-DECISION-OPTION-B-VS-BPRIME.md`
for the decision record and trigger conditions.

The current implementation (Option B) keeps two hardcoded terminators with
protocol descriptors as data. Option B' would lift the terminator itself to
a data item, making the dispatch kernel fully data-driven. This is deferred
until the trigger conditions described in the decision doc are met.
