# Dev-Tree Caveats

Notes for contributors working in the local checkout. None of these affect end-user installs.

## Single-key signing model

Every signable artifact in the dev bundle tree (`bundles/{core,standard}`) is signed with the **dev publisher key** at `.dev-keys/PUBLISHER_DEV.pem`. The trust store the test harness pins is one-entry: the dev publisher fingerprint.

## Stale bundle binaries invalidate manifests and schema support

Bundle binary directories are derived state. They are ignored by git and regenerated by `scripts/populate-bundles.sh`:

- `bundles/core/.ai/bin/<host-triple>/`
- `bundles/standard/.ai/bin/<host-triple>/`

Publishing signs and manifests the binaries that are already staged in those directories. It does not compile Rust binaries. After a source merge, a stale staged `ryeos-core-tools` can be older than the bundle descriptors it is trying to verify.

Because `bin:` resolution requires a hash match, stale binaries can fail as manifest hash mismatches. If the merge added new descriptor language, they can also fail earlier while parsing or validating schemas.

### Symptom

```
binary `ryeos-core-tools` hash mismatch: manifest declares <hash-A>,
on-disk computed <hash-B>
```

Or verification/publish fails on a newly merged schema:

```
invalid `composed_value_contract`: unknown variant `array`, expected ...
```

Or the daemon won't start because installed bundle state is stale:

```
start daemon: daemon.json never appeared at /tmp/.tmpXXX/state/daemon.json
```

### Fix

```bash
# Preferred: auto-detects drift and rebuilds
./scripts/gate.sh

# Or without running tests
./scripts/gate.sh --no-tests

# Manual
./scripts/populate-bundles.sh \
  --key .dev-keys/PUBLISHER_DEV.pem \
  --owner ryeos-dev
```

Then reinstall the refreshed bundles into the system space you use:

```bash
# Default operator system space:
target/release/ryeos init \
  --source bundles \
  --trust-file .dev-keys/PUBLISHER_DEV_TRUST.toml

# Repo-local dev system space used by scripts/dev-up.sh:
target/release/ryeos init \
  --system-space-dir .local/ryeos \
  --source bundles \
  --trust-file .dev-keys/PUBLISHER_DEV_TRUST.toml
```

Use explicit dependency roots while verifying the source tree:

```bash
target/release/ryeos-core-tools bundle-verify bundles/core --registry-root bundles/core
target/release/ryeos-core-tools bundle-verify bundles/standard --registry-root bundles/core
```

Do not manually copy one binary from `target/release/` into a bundle. That leaves CAS, manifests, sidecars, and signatures inconsistent.

If `populate-bundles.sh` still fails with `unknown variant ... expected ...` after rebuilding, the bundle descriptors have introduced a descriptor-language term that the source code itself does not support yet. Add the missing parser/composer/schema support, keep the descriptor-driven trust path, and rerun the full script. Do **not** add raw YAML scanners, hardcoded endpoint registries, or registry-sync fallbacks to work around it.

### DO NOT use `--seed 42`

The seed-42 key is NOT in the engine test trust store. Using it for bundle artifacts breaks ~130 tests with "signature not in trust store."

Recovery if you accidentally used it:
```bash
git checkout bundles/
rm -rf bundles/core/.ai/objects/blobs/*/
./scripts/populate-bundles.sh --key .dev-keys/PUBLISHER_DEV.pem --owner ryeos-dev
```

## The standard bundle

`bundles/standard/.ai/bin/<triple>/` is derived the same way as core. If standard-owned binaries are replaced, re-run the full populator so both bundles are rebuilt and re-signed in dependency order:
```bash
./scripts/populate-bundles.sh --key .dev-keys/PUBLISHER_DEV.pem --owner ryeos-dev
```

## Workspace gate

The canonical gate is `./scripts/gate.sh`. Direct `cargo nextest run` is fine but skips the auto-sync.
